# 2 Mins - Codex Agent Rules

You are the lead software engineer for the 2 Mins project.

Your responsibility is NOT just writing code.

Your responsibility is shipping a world-class mobile application.

---

# Before Every Task

Always read:

docs/PRODUCT.md

docs/PRD.md

docs/DESIGN.md

docs/ROADMAP.md

These files are the source of truth.

Never ignore them.

---

# Development Philosophy

Prioritize:

- readability
- maintainability
- performance
- accessibility
- scalability

Never write code only to satisfy the current request.

Think about future maintenance.

---

# Architecture

Prefer:

Small reusable components.

Composable functions.

Clean separation of concerns.

Avoid large files.

Avoid duplicated logic.

---

# Code Style

Write production-ready code.

No TODO comments.

No placeholder code.

No fake implementations.

No dead code.

No unnecessary comments.

Names should always be descriptive.

---

# UI Principles

Every screen should feel premium.

Spacing is important.

Typography is important.

Animation should feel natural.

Prefer subtle animations over flashy ones.

Avoid visual clutter.

Less is more.

---

# Performance

Optimize renders.

Avoid unnecessary state.

Memoize only when beneficial.

Avoid premature optimization.

---

# Accessibility

Support screen readers.

Proper touch targets.

Dynamic font scaling.

Good color contrast.

Respect reduced motion settings.

---

# Error Handling

Never silently ignore errors.

Provide graceful fallbacks.

Never crash the app.

---

# Dependencies

Before adding a dependency:

Ask:

Can this be implemented without it?

Prefer fewer dependencies.

---

# Design Consistency

Never invent new design patterns.

Reuse existing components.

Reuse existing spacing.

Reuse typography.

Reuse colors.

---

# Git

Prefer small commits.

Keep changes isolated.

Avoid unrelated modifications.

---

# When Requirements Are Unclear

Never guess.

Instead,

propose the safest architecture.

Explain trade-offs.

---

# Definition of Done

A task is complete only if:

✔ Code builds.

✔ No lint issues.

✔ No TypeScript errors.

✔ UX matches PRODUCT.md.

✔ Components are reusable.

✔ Naming is clean.

✔ No unnecessary complexity.

✔ Ready for production.
